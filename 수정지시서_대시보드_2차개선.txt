1. 관심 주식 시세 위젯 개편 (한글명 + 시총 TOP10 + 컬럼 정렬)

1-A. 국내 종목 데이터 소스를 야후 → 네이버로 교체  [중요: 단순 표시 문제가 아님]

  확인 결과(치명적):
    야후의 국내 종목 전일종가 데이터가 틀려서 등락률 자체가 오류다.
      · 삼성전자: 야후 기반 배포판 -7.72% / 네이버 실제 +3.34% (263,000원, 전일비 +8,500)
      · KOSPI 지수: 배포판 -8.28% / 네이버 실제 +49.90pt 상승
    즉 한글명 표시만 고치는 게 아니라, 국내 종목·지수는 시세 소스를 네이버로
    통째로 바꿔야 한다. (미국 종목은 야후 유지 — 문제 없음)

  사용할 네이버 API (모두 무료·무인증, 검증 완료. User-Agent는 브라우저형
  문자열 필수 — 서버의 UA 상수 대신 Mozilla/5.0 계열을 쓸 것):
    (1) 개별 종목:  https://m.stock.naver.com/api/stock/{6자리코드}/basic
        주요 필드:
          stockName                     : "삼성전자"  ← 한글명 (1-A 요구사항)
          closePrice                    : "263,000"   ← 콤마 포함 문자열. 숫자 변환 필요
          compareToPreviousClosePrice   : "8,500"     ← 전일비(절대값)
          compareToPreviousPrice.name   : "RISING"|"FALLING"|"UNCHANGED" ← 부호는 여기서 판단
          fluctuationsRatio             : "3.34"      ← 등락률(절대값). 부호 곱할 것
          marketValue                   : "15,375,713" (단위: 백만원)
          marketValueHangeul            : "1,537조 5,713억원" ← 그대로 표시 가능
          marketStatus                  : "CLOSE"|"OPEN"
    (2) 지수:  https://m.stock.naver.com/api/index/KOSPI/basic  (KOSDAQ 동일)
        필드 구조 동일. stockName="코스피".
    (3) 시총 상위:  https://m.stock.naver.com/api/stocks/marketValue/KOSPI?page=1&pageSize=10
        (KOSDAQ 버전도 동일 패턴으로 존재. 검증 완료)
        stocks[] 배열에 (1)과 같은 필드 + itemCode 포함.

  구현 지시:
    - server.py에 naver_stock(code), naver_index(name), kr_top10() 함수 추가.
      kr_top10은 KOSPI top10 + KOSDAQ top10을 합쳐 marketValue 기준 정렬 후
      상위 10개 반환 권장(사실상 KOSPI 위주가 되지만 정확성 확보).
      캐시 TTL: 개별 60초, top10 300초.
    - /api/stocks 처리부에서 심볼이 6자리 숫자(.KS/.KQ 접미 포함)면 네이버로,
      그 외(미국 티커, ^지수 중 미국계)는 기존 safe_yahoo로 분기.
      ^KS11/^KQ11은 네이버 지수 API로 매핑.
    - 응답 아이템 스키마는 기존과 동일하게 유지(name, price, changePct, marketCap …)
      + marketCapText 필드 추가(네이버 marketValueHangeul, 5-C의 축약표시 대신 사용).
    - 스파크라인·차트용 가격 히스토리는 야후 chart API를 계속 써도 됨
      (히스토리 종가는 정상, 전일종가 메타만 불량) — 3-C에서 함께 다룸.

5. 밈코인 가격 보드
--------------------------------------------------------------------------------
5-A. 코인명 옆 GMGN 링크 아이콘

  확인 결과:
    - meme_chains()가 이미 호출 중인 CoinGecko /coins/list?include_platform=true
      의 platforms 딕셔너리 "값"이 컨트랙트 주소다 (실측: pepe →
      {"ethereum":"0x6982508145454ce325ddbe47a25d4ec3d2311933", …}).
      현재 코드는 키(체인명)만 쓰고 주소를 버리고 있음 → 주소도 보존할 것.
    - GMGN URL 형식: https://gmgn.ai/{chain}/token/{address}
      체인 슬러그 매핑(이 4개만 지원 대상으로 시작):
        solana → sol / ethereum → eth / binance-smart-chain → bsc / base → base
    - 주의: GMGN은 봇 차단(curl 403)이지만 실제 브라우저에서는 정상.
      또한 네이티브 코인(DOGE 등 platforms가 빈 dict)과 GMGN 미지원 체인은
      아이콘을 생략한다.

  구현 지시:
    - server.py meme_chains(): 각 코인에 대표 체인의
      "gmgn": "https://gmgn.ai/sol/token/…" (또는 chain+address 원시 필드)
      를 추가.
    - app.js renderMeme(): 이름 옆 GMGN 아이콘 렌더(4-C와 같은
      stopPropagation/마크업 원칙, esc() 적용).

5-B. 체인 배너 재정렬 + etc 병합 + 대표 체인 단일 표기

  확인 결과(대표 체인 판별 근거):
    - CoinGecko platforms 딕셔너리의 "첫 번째 키" == asset_platform_id
      (해당 코인의 원 체인)임을 실측 확인 (pepe: 첫 키 ethereum ==
      asset_platform_id ethereum). → 별도 API 추가 호출 없이 첫 키를
      대표 체인으로 쓰면 된다.

  구현 지시(server.py meme_chains() 재작성):
    - 그룹핑 시 "모든 platforms 키"가 아니라 "첫 번째 키(대표 체인)"에만
      코인을 배정한다. (현재는 멀티체인 코인이 모든 체인 탭에 중복 등장)
    - 탭(체인 배너) 구성을 고정 순서로 변경:
        [시가총액 TOP 30] [급등 레이더] Solana | BSC | Robinhood | Base |
        Ethereum | etc
      · CoinGecko 플랫폼 id 매핑: solana, binance-smart-chain, robinhood,
        base, ethereum ("robinhood" 플랫폼 id는 기존 배포에서 6~7개 코인이
        잡히는 것을 실측 확인).
      · 표시명: binance-smart-chain → "BSC"로 축약.
      · 위 5개에 속하지 않는 모든 체인(hyperevm, arbitrum-one, avalanche,
        polygon-pos, ordinals 등)의 코인은 "etc" 탭 하나로 병합
        (내부 정렬은 시총순, 상한 20개 유지).
    - 탭이 loadMeme()마다 innerHTML로 재생성되며 스크롤이 리셋되는 기존
      문제도 이번에 함께 수정: 탭 구성이 고정되므로 최초 1회만 생성.

6. 뉴스 "데이터를 불러오지 못했습니다" (Vercel 간헐 실패)
--------------------------------------------------------------------------------
  확인 결과:
    - 2026-07-15 시점 배포판 /api/news 3종 모두 200 OK(0.3초) — 상시 장애가
      아니라 간헐 실패다.
    - 구조적 원인 3가지(모두 실측 근거 있음):
      (a) 서버가 모든 API 응답에 Cache-Control: no-store를 보냄 →
          Vercel CDN이 전혀 캐시하지 않음(X-Vercel-Cache: MISS 확인).
          모든 방문·모든 위젯 요청이 매번 서버리스 함수 → 외부 API 직행.
      (b) 서버리스 인스턴스는 메모리 캐시(CACHE/LAST_GOOD)가 인스턴스별
          독립 + 콜드스타트 시 빈 캐시. 페이지 열릴 때 위젯 8개가 동시에
          외부 API 버스트 → Google News/CoinGecko가 datacenter IP를
          간헐적으로 429/차단하면 그대로 사용자에게 에러 노출.
      (c) 외부 fetch timeout이 12초로 Vercel Hobby 함수 제한(10초)보다 김
          → 외부가 느린 날은 함수가 먼저 죽어 504.

  구현 지시:
    (1) [핵심] Handler.json()의 Cache-Control을 엔드포인트별 CDN 캐시로 교체:
          "public, s-maxage={TTL}, stale-while-revalidate=86400"
          TTL: news 600 / market·stocks 60 / coins 300 / meme-chains 300 /
               fear-greed 3600 / search·kr-top10 300
        → 같은 URL 요청이 TTL 동안 Vercel 엣지에서 바로 응답되고, TTL이
          지나도 백그라운드 재검증 동안 이전 응답을 계속 서빙(SWR)하므로
          외부 API가 일시 죽어도 사용자는 실패를 보지 않는다.
          로컬 개발 구분이 필요하면 os.environ.get("VERCEL") 유무로 분기.
    (2) fetch_json/news의 timeout을 12 → 8초로 줄이고, 실패 시 0.5초 후
        1회 재시도.
    (3) app.js fail(): 뉴스·시세 위젯이 실패하면 15~20초 후 자동 재시도
        1회(무한 재시도 금지). 에러 문구에 "잠시 후 자동 재시도" 표기.
    (4) [부하 절감] meme_chains()가 콜드스타트마다 받는
        /coins/list?include_platform=true(수 MB)가 함수 시간·메모리를 크게
        잡아먹음 → 5-B 구현 시 이 응답에서 필요한
        {meme코인 id: platforms(주소 포함)}만 추린 정적 스냅샷
        dashboard/meme_platforms.json을 scripts/로 생성·커밋하고 서버는
        그 파일을 우선 사용(없거나 미스나면 기존 API 폴백). (1)의 CDN
        캐시와 합치면 콜드스타트 문제 대부분 해소.

7. 작업 순서 제안 및 완료 기준
--------------------------------------------------------------------------------
  순서: 6(안정화) → 1-A/1-B(국내 시세 정확성) → 3-A/3-B(레이아웃)
        → 1-C(정렬) → 2(라벨 제거) → 4 → 5 → 3-C(차트 축/캔들)

  완료 기준(체크리스트):
    □ 배포판에서 삼성전자 등락률이 네이버 증권과 일치(부호 포함)
    □ 국내 탭 이름이 전부 한글(삼성전자, SK하이닉스 …)이고 시총 TOP10 자동 표시
    □ 모든 표에서 헤더-내용 세로줄 일치(모바일 포함)
    □ 화면 어디에도 W-n·WATCHLIST 류 라벨 없음
    □ 관심주식=크립토순위 너비 동일, 차트=밈보드 너비 동일,
      관심주식·차트 높이 동일
    □ 차트에 가격(y)·날짜(x) 라벨 표시, 종목/코인 클릭 시 정상 전환
    □ 순위 탭: [시가총액][L1][L2][DeFi][Robinhood]만 존재,
      코인명 옆 X 아이콘 클릭 시 공식 계정 새 탭
    □ 밈보드 체인 탭 순서 Solana→BSC→Robinhood→Base→Ethereum→etc,
      멀티체인 코인이 한 탭에만 등장, GMGN 아이콘 동작
    □ 배포판 첫 로드 연속 10회에서 뉴스 3종 위젯 에러 0회
      (X-Vercel-Cache: HIT/STALE 응답 확인)

  주의사항:
    - 외부 문자열(stockName, 코인명, twitter handle, 주소)은 innerHTML 삽입 전
      반드시 esc() 적용.
    - 행 내부에 링크 아이콘이 들어가므로 행 마크업(button)을 div로 바꿀 때
      키보드 접근성(tabindex, Enter 처리) 유지할 것.
    - 커밋은 항목별로 쪼개고, 각 커밋 후 로컬 python server.py로 8787에서
      확인 → main 푸시(Vercel 자동 배포) → 배포판 재확인.
